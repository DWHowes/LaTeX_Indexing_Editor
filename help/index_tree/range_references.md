# Range References

A **range reference** indexes a whole span of text under one heading — "see pages 42–47" — rather than a single page. In the source, it's written as a pair of `\index` macros: one marking where the range opens, one marking where it closes, with your normal document text in between.

## Inserting one

Select the text the range should cover before you insert the entry (see [Inserting Entries](../index_tree/inserting_entries.md)) — a range is created automatically whenever there's an active text selection and the entry isn't a cross-reference. With nothing selected, you get a single point reference instead.

## How it's tracked

Even though a range is two separate `\index` macros, the index tree shows it as **one** entry (the opener) — see [Viewing and Navigating](../index_tree/navigating.md). The closing macro is still there in the source and still fully tracked (renaming or deleting the entry affects both halves together), it's just not displayed as its own row.

## When ranges go wrong

Ranges are one of the more fragile parts of an index, especially in a document whose `\index` markup was generated by an automated indexing pass rather than typed by hand: a range can end up left open with no closer, closed with no opener, or a second range for the same heading can start before the first one's already-open range actually finishes. Run [Range Consistency Check](../tools/range_consistency.md) periodically to find and fix problems like these — it's built specifically for this kind of markup drift.
