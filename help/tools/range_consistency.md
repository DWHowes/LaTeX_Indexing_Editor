# Range Consistency Check

[Page ranges](../index_tree/range_references.md) are two separate `\index` macros — an opener and a closer — and nothing in that markup records which opener a given closer belongs to. When entries are generated by an automated indexing pass that re-scans a document (rather than typed by hand), it's common for that process to lose track of a range it already opened and produce markup that doesn't quite line up. **Tools → Check Range Consistency...** scans the project for exactly that kind of problem and lets you review and fix it.

## What it looks for

- **Malformed ranges** — an opener with no matching closer, or a closer with no matching opener.
- **Overlapping ranges** — two ranges for the same heading in the same file where the second one starts before the first one ends.
- **Enclosed point references** — a plain (non-range) reference for the same heading that falls inside an existing range for it — for example, an indexing pass adding a redundant single-page reference for a term that's already covered by an open range spanning that page. A styled point reference (bold, italic, etc.) inside a range is left alone, since that's a common, deliberate way to mark one page as the key discussion within a broader range, not a mistake.

Comparisons are scoped to one heading within one file — two same-named terms in different chapters are never compared against each other.

## Reviewing and applying fixes

Running the check opens a list of every problem found, grouped by category, each with the heading, file, and line(s) involved. Every item starts checked. Review the list, uncheck anything you don't want touched, then click **Apply Selected**:

- A malformed opener or closer is deleted. If it should really exist, note its location before applying and re-create the range by hand.
- Overlapping ranges are merged into a single continuous range spanning from the first one's start to the second one's end.
- An enclosed point reference is deleted.

Fixes are applied immediately to the `.tex` source and the project database — the same as any other insert or delete made through the editor, not staged for [Save](../getting_started/saving_and_closing.md). After applying, the list refreshes so you can see what's left.
